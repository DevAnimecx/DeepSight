# DeepSight v0.2.1 — Universal AI Skill Platform

## Vision

Transform DeepSight from a **Claude-only skill** into a **universal AI agent skill platform** that works across Claude (Desktop + Code) and OpenAI Codex/GPT. The installer auto-detects the user's AI platform and generates the correct configuration — no manual setup required.

---

## Theme: "Write Once, Review Anywhere"

## 1. Universal Skill Architecture

### 1.1 Multi-Platform Format

| Platform | Format | Config File | Detection |
|----------|--------|-------------|-----------|
| **Claude Desktop** | `SKILL.md` (YAML + markdown) | `claude_desktop_config.json` | Check `%APPDATA%\Claude\` or `~/Library/Application Support/Claude/` |
| **Claude Code** | `SKILL.md` + `.claude_instructions` | `.claude` file in project | Check `~/.claude/` or `$CLAUDE_CONFIG` |
| **OpenAI Codex** | Custom `codex-config.yaml` | `codex.json` in project | Check `$OPENAI_CONFIG` or `~/.config/codex/` |
| **OpenAI GPT** | GPT instructions file | `.openai/instructions.md` | Check `$OPENAI_API_KEY` presence |

### 1.2 Unified Knowledge Base

All agents, references, scripts, and tests are **shared** across platforms. Only the entry point and installation differ.

```
deepsight/
├── _universal/                 # NEW — shared across platforms
│   ├── agents/                 # 9 agents (shared)
│   ├── scripts/                # Layer 1 tools (shared)
│   ├── references/             # OWASP, arch patterns (shared)
│   ├── tests/                  # Validation fixtures (shared)
│   └── assets/                 # Templates, risk matrix (shared)
├── _platforms/                 # NEW — per-platform entry points
│   ├── claude/
│   │   └── SKILL.md            # Claude orchestrator
│   ├── openai/
│   │   └── codex-instructions.md  # Codex system prompt
│   └── gpt/
│       └── instructions.md     # GPT custom instructions
└── install/                    # NEW — universal installer
    ├── install.sh              # macOS/Linux (auto-detect)
    ├── install.ps1             # Windows (auto-detect)
    └── detect-platform.js      # Platform detection logic
```

### 1.3 Cross-Platform Agent Instructions

The 9 agents stay the same. Only the **orchestration layer** changes per platform.
Each platform gets its own `orchestrator.md` that maps the Three-Layer Protocol to platform-native features:

- **Claude**: Uses `agent.md` + tool use
- **Codex**: Uses `codex.yaml` function definitions
- **GPT**: Uses system instructions + function calling

## 2. Universal Install System

### 2.1 Auto-Detect Flow

```
                     ┌──────────────────┐
                     │  Run installer   │
                     └────────┬─────────┘
                              │
                     ┌────────▼─────────┐
                     │  Detect Platform │
                     └────────┬─────────┘
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
        ┌──────────┐   ┌──────────┐   ┌──────────────┐
        │  Claude  │   │  Codex   │   │     GPT      │
        │ Desktop  │   │   CLI    │   │  Custom GPT  │
        └────┬─────┘   └────┬─────┘   └──────┬───────┘
             │              │                │
             ▼              ▼                ▼
     ┌──────────────┐ ┌──────────┐ ┌────────────────┐
     │ Write config │ │ Write    │ │ Write system    │
     │ to Claude    │ │ codex    │ │ prompt + upload │
     │ config dir   │ │ config   │ │ instructions    │
     └──────────────┘ └──────────┘ └────────────────┘
```

### 2.2 Detection Logic

Priority chain (checks each until match):

1. **Claude Desktop**: `$APPDATA/Claude/claude_desktop_config.json` on Windows, `~/Library/Application Support/Claude/claude_desktop_config.json` on macOS
2. **Claude Code**: `~/.claude/.claude_instructions` or `$CLAUDE_CONFIG`
3. **OpenAI Codex**: `~/.config/codex/codex.json` or `$OPENAI_CONFIG`
4. **OpenAI GPT**: `$OPENAI_API_KEY` in environment, `.env`, or shell profile
5. **Fallback**: Ask user interactively

## 3. Platform-Specific Features

### 3.1 Claude (Desktop + Code) — Existing + Enhanced

- SKILL.md format with YAML header
- 9 agents loaded lazily from `agents/*.md`
- Three-Layer Depth-Charge Protocol
- Tool-based file access (Read, Grep, Glob, Bash)
- `/review`, `/audit`, `/deepsight` commands

**New in v0.2.1:**
- `.claude_instructions` file for Claude Code
- Config validation at install time
- Smoother `claude_desktop_config.json` integration

### 3.2 OpenAI Codex CLI

New format: `codex-instructions.md` with Codex-compatible tool definitions.

- Uses Codex's built-in bash, file read/write tools
- Agents as system message sections
- Function calling for Layer 1 scripts
- `codex review ./path` command

### 3.3 OpenAI GPT (Custom GPT)

New format: GPT instructions + knowledge files.

- Upload instructions as GPT system prompt
- Knowledge files: agent definitions, references
- Action configuration for external tool calls
- Works with both GPT-4 and GPT-4 Turbo

## 4. Agent Enhancements (v0.2.1)

### 4.1 New Agent: Dependency Agent

Track and audit dependency security, licensing, and freshness.

- Scans `package.json`, `requirements.txt`, `Cargo.toml`, `go.mod`, etc.
- Checks for known CVEs in direct and transitive dependencies
- Flags stale/abandoned packages
- Suggests alternatives for deprecated packages

### 4.2 Agent Instruction Upgrades

| Agent | Upgrade |
|-------|---------|
| **Security** | Add SSTI, NoSQL injection, GraphQL injection patterns |
| **Architecture** | Add microservice boundary checking, event-driven patterns |
| **Performance** | Add bundle size analysis for JS/TS, memory profiling patterns |
| **Data** | Add GDPR/CCPA compliance checks, cookie audit patterns |
| **Pattern** | Add Vue.js, Angular, Svelte guides |
| **Error** | Add circuit breaker patterns, graceful degradation |
| **Smell** | Add AI-generated code detection patterns |

### 4.3 Cross-Platform Output Format

Unified output schema that works across all platforms:

```json
{
  "platform": "claude | codex | gpt",
  "version": "0.2.1",
  "review": {
    "risk_score": 7,
    "verdict": "request_changes",
    "findings": [
      {
        "file": "src/auth/login.ts",
        "line": 42,
        "severity": "critical",
        "agent": "security",
        "finding": "SQL injection via string concatenation",
        "fix": "Use parameterized query: db.query('SELECT * FROM users WHERE id = ?', [userId])"
      }
    ]
  }
}
```

## 7. Migration Path

Users of v0.1.x can upgrade seamlessly:

1. Existing `~/.agents/skills/deepsight/` is detected and updated
2. New platform files are added alongside existing config
3. No loss of custom per-project team-playbook.md settings
4. `install --update` flag for in-place upgrade
